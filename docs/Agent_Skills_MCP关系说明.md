# Agent、Skills 和 MCP 关系说明

## 概述

本文档解释在**情感聊天机器人项目**中，Agent、Skills 和 MCP 三者之间的关系和各自的作用。

---

## 一、三者定义

### 1. Agent（智能体）

**在项目中的定义**：
- Agent 是智能体系统的核心，负责处理用户输入并生成回复
- 包含多个模块：
  - **Planner（规划器）**：分析用户意图，制定执行计划
  - **Tool Caller（工具调用器）**：执行具体的工具调用（如搜索记忆、查询知识库等）
  - **Reflector（反思器）**：评估交互效果，优化后续响应
  - **Agent Core（核心协调器）**：协调所有模块，实现完整的处理流程


### 2. Skills（技能）

**在项目中的定义**：
- 项目本身不包含 Skills 系统
- Skills 是开发工具的功能，不是项目运行时的一部分

**Skills 的实际价值**：
虽然 Skills 不是项目运行时的一部分，但在开发过程中具有重要价值：

1. **标准化开发流程**：
   - 通过 Skills 可以创建项目特定的开发规则和编码规范
   - 例如：为情感聊天项目创建 Python 编码规范、API 设计规范等
   - 确保团队开发的一致性和代码质量

2. **提升开发效率**：
   - Skills 可以指导 AI 助手理解项目架构和代码结构
   - 帮助快速定位和理解 Agent Core、Planner、Tool Caller、Reflector 等模块的关系
   - 加速代码审查、重构和优化工作

3. **知识传承**：
   - 将项目特定的开发经验、最佳实践封装为 Skills
   - 新成员可以通过 Skills 快速了解项目架构和开发规范
   - 减少重复性的解释和培训工作

### 3. MCP（Model Context Protocol）

**在项目中的定义**（项目内部 MCP）：
- MCP 是项目**自定义的通信协议**，用于 Agent 系统内部模块间的标准化通信
- 定义在 `backend/modules/agent/protocol/mcp.py`
- 提供标准化的消息格式，包含：
  - `content`：自然语言内容
  - `context`：结构化上下文（用户画像、情感状态、记忆摘要等）
  - `tool_calls`：工具调用指令
  - `tool_responses`：工具执行结果
- 实现模块间的低耦合、高内聚通信

## 三、详细关系说明

### 1. Agent 与 Skills 的关系

**当前状态**：
- 项目本身不包含 Skills 系统
- Skills 目前是开发工具的功能，与项目运行时无关

**未来规划**：
- 计划在项目中引入 Skills 系统，作为 Agent 的能力扩展机制
- Skills 可以让 Agent 具备更专业的能力，例如：
  - **情感支持技能**：针对不同情感状态的专业支持策略
  - **危机干预技能**：识别和处理危机情况的标准化流程
  - **个性化回复技能**：基于用户画像的定制化回复策略
  - **知识检索技能**：智能调用 RAG 知识库的技能
- Skills 将通过项目内部 MCP 协议与 Agent Core 集成
- 实现方式：在 `backend/modules/agent/` 下创建 `skills/` 模块

### 2. Agent 与 MCP 的关系

**在项目中**：
- **Agent Core** 使用**项目内部 MCP 协议**来协调各个模块
- 通信流程：
  ```
  用户输入
    ↓
  Agent Core (创建 user_input MCP 消息)
    ↓
  Planner (接收 MCP 消息，输出 planner_output)
    ↓
  Tool Caller (接收 MCP 消息，执行工具，输出 tool_response)
    ↓
  Agent Core (生成回复，创建 agent_response MCP 消息)
    ↓
  Reflector (接收 MCP 消息，评估，输出 evaluation)
  ```

### 3. Skills 与 MCP 的关系

**在项目中**：
- 项目内部 MCP 协议与 Skills 没有直接关系
- Skills 是开发工具的功能，不是项目运行时的一部分

## 四、实际应用场景

### 场景 1：项目运行时

```
1. 用户通过前端发送消息："我最近心情不好"
2. 项目后端接收请求
3. Agent Core 使用项目内部 MCP 协议：
   - 创建 user_input MCP 消息
   - 传递给 Planner 模块
   - Planner 分析意图，返回 planner_output MCP 消息
   - Tool Caller 执行工具（如情感分析），返回 tool_response
   - Agent Core 生成回复，返回 agent_response
4. 返回回复给用户
```

## 五、总结

| 概念 | 作用域 | 主要功能 | 关系 |
|------|--------|---------|------|
| **Agent** | 项目 | 智能体核心，处理用户输入并生成回复 | 使用项目内部 MCP 协议 |
| **Skills** | 开发工具 | 开发工具功能，非项目运行时功能 | 与项目运行时无关 |
| **MCP** | 项目 | 项目内部模块间通信协议 | 被 Agent Core 使用，协调各模块 |

### 核心关系

1. **Agent 是执行者**：项目中的 Agent Core 是智能体系统的核心，负责处理用户输入并生成回复

2. **Skills 是开发工具功能**：Skills 是开发工具的功能，不是项目运行时的一部分

3. **MCP 是桥梁**：
   - **项目内部 MCP**：连接 Agent 系统的各个模块（Planner、Tool Caller、Reflector），实现标准化通信

4. **三者协作**：
   - 项目中：Agent Core 使用项目内部 MCP 协议 → 协调各模块（Planner、Tool Caller、Reflector）→ 处理用户请求
