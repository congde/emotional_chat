# PRD：Codex 风格情感陪伴工作台

## 1. 文档信息

- 产品名称：心语情感陪伴工作台
- 版本：V2.0
- 产品形态：Web 桌面优先，响应式支持移动端
- 视觉方向：Codex 式低噪声工作台
- 文档状态：规划稿

## 2. 背景与问题

当前前端能够完成聊天、历史、技能、个性化、附件和反馈，但界面仍是传统“侧栏 + 聊天框”形态，存在以下问题：

- 功能入口层级不清，历史、技能、个性化主要依靠按钮和弹窗。
- 页面缺少稳定的信息分区，当前情绪与相关记忆没有持续可见的承载区域。
- 设计语言不统一，组件内存在分散颜色、透明背景和未定义 CSS 变量。
- `App.js` 同时协调多套 session 状态，流式完成与历史刷新不同步。
- 移动端仅隐藏侧栏，缺少完整导航替代方案。

## 3. 产品目标

### 3.1 用户目标

- 用户打开页面后 3 秒内理解“可以开始一段对话”。
- 用户能够快速找回历史会话，并理解系统记住了什么。
- 用户能在不中断聊天的情况下查看情绪、记忆和建议。
- 高级能力可发现，但不会压迫主聊天路径。

### 3.2 业务目标

- 提升新用户首条消息发送率。
- 提升历史会话恢复使用率。
- 提升记忆与技能功能的可发现性。
- 降低因布局、透明度和状态不同步造成的体验缺陷。

### 3.3 非目标

- 不将产品改造成 IDE 或代码开发工具。
- 不在本期重写后端技术栈。
- 不在本期增加团队、多租户或付费体系。
- 不复制 Codex 的商标、图标或专属资产，仅参考布局原则。

## 4. 目标用户与场景

### 用户 A：需要即时倾诉

希望打开页面后立即表达情绪，不想先配置复杂选项。

### 用户 B：持续记录情绪

希望跨会话回顾历史、情绪趋势和长期记忆。

### 用户 C：主动探索工具

希望使用冥想、评估、日程提醒或资源推荐等技能。

## 5. 核心用户旅程

### 5.1 首次对话

1. 用户进入工作台。
2. 主区展示欢迎语、能力说明和输入器。
3. 用户输入消息并发送。
4. 系统显示连接状态并开始流式输出。
5. 完成后历史列表出现新会话，右栏展示情绪和相关记忆。

### 5.2 恢复历史对话

1. 用户从左栏选择历史会话或搜索关键词。
2. 主区加载完整消息并定位到底部。
3. 右栏切换至该会话的情绪和记忆上下文。
4. 用户继续发送，新消息写回同一会话。

### 5.3 使用技能

1. 用户打开“技能”。
2. 按分类或关键词筛选。
3. 选择技能后回到 Composer，并生成可编辑的调用提示。
4. 用户确认发送，结果进入当前会话。

## 6. 功能需求

### 6.1 应用外壳（P0）

- 桌面端采用左导航、主工作区、右上下文栏三栏布局。
- 左栏宽度建议 248px；右栏建议 300px；主区自适应。
- 左右栏支持独立折叠，并记忆用户偏好。
- 顶部提供当前会话标题和最小必要操作。
- 主工作区不使用大面积渐变或高透明玻璃效果。

验收：1440px 宽度下三栏同时可用，主消息区域不小于 640px。

### 6.2 左侧导航（P0）

- 展示品牌、搜索、新对话。
- 一级入口：历史对话、记忆、技能、个性化。
- 历史会话按时间分组，展示标题和相对时间。
- Hover 显示更多菜单：重命名、删除。
- 底部展示用户标识、主题切换和设置。

验收：新建、选择和删除会话后，选中态与主区内容一致。

### 6.3 主聊天区（P0）

- 空状态标题建议：“今天想聊些什么？”
- 消息正文最大宽度建议 760px，适配长文本阅读。
- 用户与 AI 消息通过对齐、头像或背景轻量区分，避免厚重气泡。
- 支持 Markdown、列表、引用、链接和代码块。
- AI 消息提供复制、重试、反馈操作。
- 流式阶段显示光标和“停止生成”。

验收：流式失败时保留用户消息，展示可重试错误卡，不追加重复消息。

### 6.4 浮动输入器 Composer（P0）

- 固定在主区底部并随内容高度自动增长，最大高度 200px。
- 左侧入口：附件、技能或更多能力。
- 底部状态：当前模式、深度思考开关。
- 右侧入口：语音与发送/停止按钮。
- `Enter` 发送，`Shift+Enter` 换行；输入法组合期间不得误发送。
- 附件和 URL 以紧凑 chip 展示，可单独移除。

验收：键盘、粘贴文件、附件上传和取消生成均可用。

### 6.5 右侧上下文栏（P1）

- 情绪卡：当前情绪、强度和最近趋势。
- 记忆卡：本轮相关记忆摘要、类型和重要性。
- 建议卡：最多三条，可点击填入输入框。
- 上下文卡：当前附件和解析链接。
- 支持整体折叠；无内容时不显示空白大卡片。

验收：切换会话时右栏同时更新，不显示上一用户或上一会话的数据。

### 6.6 历史管理（P0）

- 支持列表、搜索、选择、删除和批量删除。
- 会话标题默认取首条用户消息，可后续重命名。
- 首次流式完成后立即出现在左栏，无需刷新页面。
- 会话继续对话后更新排序和预览。

验收：刷新页面、切换会话和继续聊天后，消息数量与时间顺序正确。

### 6.7 记忆中心（P1）

- 支持记忆列表、类型筛选、语义搜索和统计摘要。
- 支持修改重要性与删除，危险操作需确认。
- 清楚说明“为什么记住”和“如何使用”。
- 不直接暴露内部向量数据库概念。

验收：越权删除失败；更新重要性后列表和检索结果保持一致。

### 6.8 技能中心（P1）

- 支持全部、情绪、记忆、资源、评估、日程分类。
- 分类标签文字必须在边框内水平、垂直居中。
- 支持搜索、导入和手动添加。
- 选择后生成可编辑内容，不直接执行高影响操作。

### 6.9 个性化（P1）

- 角色、表达风格、高级设置分组呈现。
- 面板使用完全不透明背景，确保文字可读。
- 保存时显示进行中、成功和失败状态。
- 关闭未保存修改时提供确认。

### 6.10 响应式（P1）

- ≥1200px：默认三栏。
- 768–1199px：右栏默认收起，通过按钮打开抽屉。
- <768px：左栏和右栏均使用全高抽屉，主区保留聊天与输入器。
- 避免仅通过 `display: none` 移除唯一功能入口。

## 7. 视觉规范

### 7.1 设计语言

- 克制、安静、可信任，优先内容而非装饰。
- 页面背景使用低饱和冷灰；核心面板使用实色背景。
- 主强调色只用于选中、发送和关键操作。
- 边框优先于重阴影；动画时长建议 120–220ms。

### 7.2 建议令牌

| Token | 浅色 | 深色 |
|---|---|---|
| `--bg-app` | `#f4f6f8` | `#0f1115` |
| `--bg-panel` | `#ffffff` | `#181b20` |
| `--bg-subtle` | `#eef2f5` | `#22262d` |
| `--text-primary` | `#202123` | `#f2f3f5` |
| `--text-secondary` | `#667085` | `#a6adb8` |
| `--border` | `#e3e7eb` | `#30353d` |
| `--accent` | `#2563eb` | `#60a5fa` |

圆角层级：控件 8px、卡片 12px、Composer 18px、Dialog 20px。

## 8. 状态与异常

- Loading：使用局部 skeleton，不阻塞整个应用。
- Empty：提供下一步动作，不只显示“暂无数据”。
- Error：错误靠近发生区域，提供重试；认证失败给出配置提示但不暴露密钥。
- Offline：保留输入草稿，恢复连接后允许手动重发。
- Streaming cancelled：保留已生成文本，并标记“已停止”。

## 9. 数据与接口依赖

沿用现有接口：

- `/chat/stream`
- `/chat/users/{user_id}/sessions`
- `/chat/sessions/{session_id}/history`
- `/chat/sessions/{session_id}`
- `/memory/users/{user_id}/memories`
- `/memory/users/{user_id}/memories/search`
- `/memory/users/{user_id}/statistics`
- `/memory/users/{user_id}/profile`

建议补充或修正：

- 流式 `done` 后统一返回 session 元数据。
- 每次保存消息时更新 `ChatSession.updated_at`。
- 会话重命名接口。
- 当前会话相关记忆与情绪上下文聚合接口，减少右栏多次请求。

## 10. 非功能要求

- 首屏可交互目标：本地环境 < 2 秒。
- 流式请求发起后 300ms 内显示连接反馈。
- 主要交互满足 WCAG AA 对比度。
- 所有图标按钮有可访问名称和 Tooltip。
- 不在前端日志中输出 API Key、完整隐私内容或敏感记忆。

## 11. 指标

- 首条消息发送率。
- 历史会话恢复率。
- 新会话完成后历史即时出现成功率。
- 消息生成失败率与重试成功率。
- 记忆中心和技能中心访问率。
- 前端异常数、布局回归数和无障碍缺陷数。
